Scan-Typen vergleichen
SYN Scan (-sS)
  • Weniger Logs als Connect Scan (Verbindung wird nie aufgebaut)
  • ABER: IDS (Suricata) erkennt SYN Scan Patterns
  • auth.log zeigt NICHTS — kein Login-Versuch
  • Firewall-Log zeigt: viele SYN-Pakete von einer IP an verschiedene Ports
alert tcp any any -> $HOME_NET any (msg:"SCAN nmap SYN"; flags:S; threshold:type both,track by_src,count 20,seconds 5; sid:2000001;)
Connect Scan (-sT)
  • VOLLER Handshake = normaler Verbindungsaufbau
  • Hinterlässt Logs (access.log, auth.log)
  • Einfacher zu erkennen, aber sieht "normaler" aus
XMAS / NULL Scan
  • Setzt ungewöhnliche TCP-Flags (FIN+PSH+URG oder keine)
  • IDS erkennt das sofort — kein normaler Traffic hat diese Flags
  • Wird von modernen Firewalls geblockt
  • Ambiguität: offene und gefilterte Ports beide zeigen keine Antwort
