Setup-Schritte
BIOS: Virtualisierung aktivieren
AMD-V (SVM Mode) im BIOS einschalten. Bei Ryzen: BIOS → Advanced → CPU → SVM Mode → Enabled
BIOSAMD-V
Hyper-V aktivieren
Windows-Feature einschalten. PowerShell als Admin: Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
WindowsHyper-V
Proxmox VM erstellen
Proxmox ISO als nested VM unter Hyper-V. 8 Cores, 16 GB RAM, 120 GB Disk. Nested Virtualization per PowerShell aktivieren.
Proxmoxnested virt
IP-Plan & Subnetting
Netzwerkbereiche festlegen: Management, Services, Lab, DMZ. RFC1918 Ranges planen.
NetzwerkIPSubnetting
Erste Linux VM
Ubuntu Server 24.04 LTS in Proxmox. SSH einrichten, Grundkonfiguration, Updates.
LinuxUbuntuSSH
Strategie: Proxmox läuft als VM unter Hyper-V (nested virtualization). Windows 11 bleibt unangetastet als Host. Ryzen 5800X unterstützt AMD-V nativ — nested virt funktioniert gut. Ressourcen sind dynamisch anpassbar, also lieber jetzt konservativ starten (8C / 16 GB) und später hochskalieren.
PHASE 2
Netz & Erreichbarkeit
VPN · Reverse Proxy · DNS · VLANs · Firewall
▾
WireGuard VPN
Homelab von außen erreichbar machen. Eigener VPN-Endpunkt, sichere Verbindung von unterwegs.
VPNWireGuardSecurity
Reverse Proxy
Nginx Proxy Manager oder Traefik. Dienste über Subdomains erreichbar, TLS-Terminierung.
NginxTraefikTLS
Interner DNS
Pi-hole oder AdGuard Home. Interne Hostnamen auflösen, DNS-Filter, lokale Domains.
DNSPi-holeAdGuard
VLANs konfigurieren
Netzwerksegmentierung: Management VLAN, Service VLAN, Lab VLAN. Isolation zwischen Bereichen.
VLANSwitchIsolation
pfSense / OPNsense
Virtuelle Firewall-VM in Proxmox. Routing zwischen VLANs, Firewall-Regeln, NAT.
FirewallpfSenseRouting
Dynamisches DNS (DDNS)
Externe Erreichbarkeit trotz wechselnder Heim-IP. Cloudflare DDNS oder DuckDNS.
DDNSCloudflareDNS
